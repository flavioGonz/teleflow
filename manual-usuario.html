filter_list Búsqueda y Filtros

1. Buscar por Nombre o Interno

En el campo de búsqueda, escribe:

  • 1001 - Para encontrar el agente del interno 1001
  • Juan - Para encontrar al agente llamado Juan
  • 192.168 - Para encontrar agentes con esa IP

2. Filtrar por Estado

Usa el dropdown desplegable:

  • Todos los Estados - Muestra todos los agentes
  • Online - Solo agentes disponibles
  • En Llamada - Solo los que están hablando ahora
  • Offline - Solo los desconectados

3. Actualizar Manualmente

Haz clic en el botón 🔄 Actualizar para refrescar los datos al instante.

✓ Sabías que: El dashboard se actualiza automáticamente cada 5 segundos. El botón de actualizar es para cuando necesitas ver cambios inmediatos.

person Ver Detalles de Agente

Haz clic en cualquier fila de agente para abrir un modal con información detallada.

Información que Verás:

Información Técnica

  • Nombre: Del agente
  • Estado: Actual (Online, Busy, etc.)
  • IP Origen: De donde se conecta
  • MAC: Dirección MAC del dispositivo
  • RTT: Latencia de red

Métricas del Día

  • Llamadas Manejadas: Total del día
  • AHT Promedio: Tiempo por llamada
  • ACW Promedio: After Call Work (trabajo después)
  • En Llamada: Si está hablando, duración

Acciones desde el Detalle:

Esto envía una notification de prueba a tu navegador para verificar que las notificaciones funcionan.

notifications_active Notificaciones PWA

Las notificaciones te alertan cuando ocurren eventos importantes sin necesidad de tener el dashboard abierto.

Pasos para Habilitar Notificaciones:

  1. Haz clic en el icono ⚙️ (arriba a la derecha del dashboard)
  2. Verás el panel "Configuración" con varios toggles
  3. Activa los tipos de notificaciones que deseas recibir:
    • 🔓 Agente Conectado
    • 🔒 Agente Desconectado
    • 🔄 Cambio de Estado
    • 📞 Llamada Recibida
    • ❌ Llamada Perdida
    • ⚠️ Alerta de Cola
    • ☕ Agente en Break
  4. Haz clic en "🔔 Habilitar Notificaciones"
  5. Autoriza en el popup del navegador que aparecerá

ℹ️ Importante: Las notificaciones requieren HTTPS (protocolo seguro). Si accedes por HTTP, las notificaciones no funcionarán.

✓ Notificaciones Guardadas: Tus preferencias se guardan automáticamente en el navegador. La próxima vez que abras el dashboard, tendrás la misma configuración.

Tipos de Notificaciones:

Agente Conectado:

Se recibe cuando un agente se conecta a su estación

Agente Desconectado:

Se recibe cuando un agente cierra su sesión

Cambio de Estado:

Cuando pasa de disponible a busy, en break, etc.

Llamada Recibida:

Cuando un agente recibe una llamada entrante

Alerta de Cola:

Cuando hay demasiadas llamadas esperando

build Solucionar Problemas

❌ No puedo acceder al dashboard

Posibles causas:

  • No estás conectado a la red corporativa (usa VPN)
  • El servidor está caído (contacta al administrador)
  • La URL tiene un error tipográfico

Solución: Prueba con https://pbx01.infratec.com.uy primero.

❌ No veo agentes en la lista

Posibles causas:

  • Los agentes aún no se han conectado hoy
  • Hay un problema con Asterisk
  • Filtros aplicados muy restrictivos

Solución: Limpia los filtros y haz clic en "🔄 Actualizar".

❌ Las notificaciones no funcionan

Requisitos:

  • ✓ Debes estar en HTTPS (no HTTP)
  • ✓ Has hecho clic en "🔔 Habilitar Notificaciones"
  • ✓ Permitiste notificaciones en el navegador
  • ✓ El toggle de tipo está activado

Solución: Revisa cada uno de estos puntos en orden.

❌ Los datos se ven antiguos

El dashboard actualiza cada 5 segundos, pero a veces:

  • Tu navegador está cacheando (borra caché)
  • La conexión de red es lenta
  • El servidor está sobrecargado

Solución: Haz clic en "🔄 Actualizar" para refrescar inmediatamente.

❌ Las métricas (AHT, Llamadas) son 00:00

Esto puede significa que:

  • Asterisk aún no ha registrado datos del agente
  • La base de datos de Asterisk está vacía
  • El agente es nuevo

Solución: Los datos aparecerán cuando el agente termine una llamada.

👨‍💼 Contactar Soporte: Si el problema persiste, reporta a tu administrador de IISABEL con:

  • Horario exacto del problema
  • URL que usaste
  • Captura de pantalla del error
  • Tu navegador y versión
